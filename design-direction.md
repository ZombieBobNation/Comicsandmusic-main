# Design Direction

## Project Mood

The website should match the tone of the music style used in the project: fast, aggressive, tense, cathartic, and chaotic-but-controlled.

The site should not feel like a plain school project. It should feel like a comic/music AI case study with enough intensity to match the work, but enough structure that people understand the project and trust the process.

The main feeling should be:

**This person knows what they are talking about, and this makes sense.**

The design should feel intense, but still clear and usable.

## Color Direction

Use a balanced dark/light direction. The site should include dark sections, but it should not be fully black or hard to read.

Approved color direction:

- black
- teal
- purple
- toxic neon yellow-green from the uploaded logo
- white/off-white only when needed for readability

The color palette should feel connected to comics, heavy metal, AI, and the project brand.

The toxic neon logo color can be used as an accent, but it should not turn into regular yellow. It should feel more acidic, glowing, or radioactive.

## Typography Direction

Use a mix between **heavy metal** and **comic book** typography.

Heading style:

- loud
- bold
- aggressive
- sharp
- comic/metal inspired
- strong title treatments
- possible distressed or inked styling, but still readable

Body text:

- clean enough for a portfolio case study
- easy to read
- not overly decorative
- should support long project explanations without becoming exhausting

The typography should make the project feel energetic, but the information still needs to be understandable.

## Layout Direction

Use a **hybrid case study layout**.

The site should have the structure of a clear portfolio case study, but the visual treatment should borrow from comic panels, album covers, and research/evidence cards.

Suggested navigation:

- Overview
- Process
- Evidence
- Final Work
- Reflection

The layout should use:

- stacked sections for clarity
- comic-style panels where useful
- cards for evidence blocks
- before/after comparisons
- tool cards
- process timeline sections where the project changes over time
- clear section breaks

The layout should feel like **controlled chaos**, not full chaos. It can be sharp, energetic, and visual, but it should still be easy to follow.

## Image / Media Direction

The homepage should open with a strong visual hook.

Preferred hero direction:

**Comic splash page**

The hero should feel dramatic and comic-inspired, with heavy outlines or panel energy. It should immediately connect the viewer to the comic/music nature of the project.

The opening media should show an example of each side of the project:

- comic panels
- album art

A carousel or paired visual preview could work well. The site should start visually, then explain how AI helped in both fields, then move into process evidence.

Media to include:

- comic panels
- album covers
- black-and-white panel to colored panel comparison
- wrong AI output versus corrected AI output
- prompt notes or prompt archive screenshots
- Spotify/iTunes/Apple Music publishing links or screenshots
- Milanote board visuals/resources if useful

## References

General visual references:

- superhero comics
- monster comics
- comic book splash pages
- heavy metal album covers from any year
- Blink-182 style music culture
- Falling in Reverse style music culture
- album cover energy
- bold comic panel composition

The references should guide the tone, not be copied directly.

The site should feel like it belongs between a comic process board, a metal album release, and a serious portfolio case study.

## Avoid

Avoid:

- pink
- red
- regular yellow
- pastel colors
- dull colors
- bland corporate colors
- overly corporate design
- overly academic/plain design
- too much AI glitch
- too much grunge/noise
- overly clean Apple-like design
- childish cartoon styling
- cluttered layouts
- effects that make the text hard to read
- a site that feels like random AI outputs without human control

Use texture carefully. Comic and metal texture are approved, but they should not take over the site.

## Notes for the Coding Agent

Build the site as a clear portfolio case study with a strong comic/metal visual identity.

Priority:

**Balanced: clear case study with strong comic/metal visual identity.**

Do not make the site too experimental or hard to navigate. The user wants people to understand the project and believe the process makes sense.

Recommended structure:

1. **Hero / Overview**
   - comic splash page feel
   - bold project title
   - short summary
   - visual carousel or preview strip showing comic panels and album art

2. **Project Question**
   - explain the main inquiry: using AI as an assistant while staying in control

3. **Process**
   - show comic workflow and music workflow
   - use stacked sections, cards, and process/timeline blocks

4. **Evidence**
   - before/after outputs
   - wrong result versus corrected result
   - prompt archive examples
   - publishing evidence

5. **Final Work**
   - comic sample
   - music releases
   - album/single links

6. **Reflection**
   - public reflection only
   - keep private ethical/guilt/resume concerns out unless intentionally edited for public use

Visual system notes:

- Use black, teal, purple, and toxic neon yellow-green.
- Use dark sections mixed with readable lighter sections.
- Use comic panel borders, halftones, ink marks, scratches, and distressed edges in controlled amounts.
- Use aggressive display typography for headings.
- Use clean readable body typography.
- Keep navigation simple: Overview, Process, Evidence, Final Work, Reflection.
- Avoid making the site look corporate, childish, overly academic, or messy.
